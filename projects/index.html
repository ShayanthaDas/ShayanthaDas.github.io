Database Architecture

E-Commerce Multi-Tier Cohort Retention Engine

PostgreSQLMySQLPython (Pandas)Google Colab

Business Context & Objective

A high-volume consumer platform experienced unmapped transaction dropouts between digital cart insertions and final checkouts. The task was to parse over 1.2 million raw transactional lines to flag bottleneck coordinates and isolate user retention trends across consecutive 30-day cohorts.

Technical Execution Layer

Engineered recursive Common Table Expressions (CTEs) and multi-stage queries inside PostgreSQL. Applied window tracking formulations (`LEAD`, `LAG`, and `FIRST_VALUE`) to dynamically group consumer segments.

POSTGRESQL PROFILE // QUERY SCHEDULERACTIVE RUN
WITH MonthlyUserActivity AS (
    SELECT user_id, DATE_TRUNC('month', transaction_date) AS activity_month,
           LAG(DATE_TRUNC('month', transaction_date), 1) OVER (PARTITION BY user_id ORDER BY transaction_date) AS previous_month
    FROM source_transaction_ledger
)
SELECT activity_month, COUNT(DISTINCT user_id) AS active_cohort_count
FROM MonthlyUserActivity WHERE activity_month = previous_month + INTERVAL '1 month'
GROUP BY 1 ORDER BY 1;
